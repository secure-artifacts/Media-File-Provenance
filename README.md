> 基于 perceptual hash (phash) 的素材追踪与溯源工具  
> 支持图片 / 视频的跨平台分发追踪，记录多级制作关系

## 环境要求

| 依赖 | 版本 |
|------|------|
| Python | 3.10+ |
| MySQL | 5.7 / 8.x |

```bash
pip install PyQt6 opencv-python pymysql Pillow imagehash piexif mutagen
```

## 功能说明

### Tab 1 · 素材登记

将原始素材（图片 / 视频）拖入，点击执行。系统自动：
- 计算 phash（感知哈希，16位十六进制）
- 写入文件的 Windows「备注」字段（exiftool）
- 存入数据库 `assets` 表

### Tab 2 · 衍生登记（Derive）

记录「A 素材 → 处理后 → B 素材」的单对单关系。  
拖入来源文件 + 衍生文件，选择关系类型，执行后：
- 数据库写入 `rel_derive` 关系记录
- B 文件的备注字段写入：`from=A的phash(A的制作人)>A的父级phash(...)>...`

### Tab 3 · 成品封装（Compose）

记录「多个素材 → 合成 → 一个成品」的多对一关系。  
拖入所有组件文件 + 成品文件，执行后：
- 数据库写入 `rel_compose` 关系记录
- 成品文件备注写入：
  ```
  parts=组件phash(制作人)>[子组件phash(制作人)>祖先phash(制作人)], 另一组件phash(制作人)
  ```

### Tab 4 · Canva 模板登记

记录「哪些素材被用于 Canva 模板」。  
拖入使用的素材文件，填写模板名称，生成带时间戳的模板 ID，可一键复制。

### Tab 5 · 溯源查询

**按文件查询：** 拖入任意文件 →  
- 若有数据库记录：展示完整溯源树（多级衍生 + 封装组件层级）  
- 若无记录但哈希相近：模糊匹配后展示关联信息  

**按 Canva ID 查询：** 输入模板 ID → 查出该模板使用了哪些素材及其完整来源链。

**溯源树结构示例：**
```
🎯 最终成品视频
├── 📦 组件来源
│   ├── 📄 修图版图片.png（制作人：小强）
│   │   └── ⬆ 衍生来源
│   │       └── 📄 原始图片.png（制作人：张三）
│   └── 📄 背景视频.mp4（制作人：李四）
│       └── 🔧 子组件（该素材本身也是封装品）
│           ├── 📄 片段A.mp4
│           └── 📄 片段B.mp4
└── 🎨 Canva模板 (1个)
    └── 🎨 春季推广模板
```

### Tab 6 · 素材库

浏览全量素材，支持按文件名 / 作者搜索。

## 使用流程

典型的三阶段流程：

```
[生图组] 生成图片 → Tab1 素材登记
      ↓
[修图组] 处理图片 → Tab2 衍生登记（原图 → 修图）
      ↓
[Canva组] 合成视频 → Tab3 成品封装（修图+BGM+视频片段 → 成品）
                  → Tab4 登记 Canva 模板ID
```

## 元数据格式说明

写入文件「Windows 备注」字段的格式：

**原始素材：**
```
phash=a1b2c3d4e5f60718; by=张三; date=2026-03-13
```

**衍生素材：**
```
phash=b2c3d4e5f6071819; by=李四; date=2026-03-13; from=a1b2c3d4(张三)>祖先phash(王五)
```

**封装成品：**
```
phash=cfbf71046a6d8291; by=小强; date=2026-03-13; parts=组件A的phash(制作人)>[子组件phash(制作人)],组件B的phash(制作人)
```

> `>` 表示衍生自，`>[...]` 表示由方括号内的多个组件封装而来。  
> 文件本身即携带完整溯源档案，离开数据库也可人工解读。

## 下一步开发计划

### 🗂️ P1 · 文件夹批量扫描入库

**背景：** 现有几百个素材文件未登记，需要一次性全部抓取到数据库。

**方案：**
- 新增「批量扫描」模块
- 输入一个文件夹路径，递归遍历所有图片/视频
- 对每个文件计算 phash，写入数据库（已有则跳过，避免重复）
- 显示进度条 + 扫描报告（新增 N 个 / 已有 N 个 / 失败 N 个）

**实现位置：** 新 Tab 或独立入口，调用现有 `ensure_registered()`  
**优先级：** ⭐⭐⭐⭐⭐ 高

---

### 📁 P2 · Canva 文件夹自动关联模块

**背景：** Canva 下载的文件夹，文件名通常含有模板 ID，需要：
1. 解析文件夹名 / 文件名中的 Canva 模板 ID
2. 计算文件夹内每个视频/图片的 phash
3. 在数据库中匹配已有素材（模糊查询）
4. 自动建立「素材 → Canva 模板」关联记录

**方案：**
```
用户拖入 Canva 下载文件夹
    ↓
解析文件夹名提取模板ID（或手动输入）
    ↓
遍历文件夹内所有媒体文件
    ↓
计算 phash → 模糊匹配数据库素材
    ↓
展示匹配结果（用户确认）
    ↓
写入 canva_templates 表 + rel_compose 关联
```

**需要讨论的点：**
- Canva 文件夹的命名规律？（文件名格式示例截图）
- 视频被压缩/转码后 phash 变化多大？（需实测阈值）
- 模糊匹配阈值：目前 ≤12，Canva 输出是否需要调整？

**优先级：** ⭐⭐⭐⭐ 高

---

### 🔬 P3 · 哈希稳定性测试（特别是 PNG）

**背景：** PNG 经过不同软件保存、压缩、平台上传后 phash 是否还能匹配？

**待测试场景：**

| 操作 | 预期 phash 变化 | 需要测试 |
|------|----------------|---------|
| 原图直接复制 | 0（完全一致） | ✅ |
| PNG → 另存为 PNG | 0~2 | ✅ |
| PNG → 微信发送接收 | 4~8 | ✅ |
| PNG → 上传微博/小红书 | 8~15 | ✅ |
| PNG 被截图 | 10~20（依截图范围） | ✅ |
| PNG → 压缩 → PNG | 2~6 | ✅ |
| 加水印后 | 2~8 | ✅ |
| 图片被裁剪 | >20（失效） | 需记录边界 |

**测试方法：**
1. 取 5 张已登记的原始 PNG
2. 分别经过上述处理
3. 拖入系统溯源查询
4. 记录每种操作的汉明距离
5. 调整 `threshold` 参数（目前 12）

**优先级：** ⭐⭐⭐⭐ 高（直接影响溯源可靠性）

### ⚡ P4 · 性能与体验优化

- **缩略图缓存**：库页面频繁刷新时重复生成缩略图，考虑本地磁盘缓存
- **批量操作进度**：大文件夹扫描时进度条 + 可取消
- **导出报告**：溯源树导出为 PDF / Excel，方便存档
- **搜索增强**：全量库支持按 phash 前缀搜索、按日期范围过滤
