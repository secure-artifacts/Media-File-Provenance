## 👥 各团队快速指南

### 📸 生图组 - 原始素材登记

**目标**: 记录生成的图片

**操作步骤**:
1. 打开软件，切换到 **"素材登记"** 表签
2. 拖入 `generated_image_1.png`, `generated_image_2.png` ...
3. 点击 **"执行批量登记"** 
4. 查看日志确认所有图片已登记 ✅

**系统自动记录**:
- ✅ 图片的phash值
- ✅ 文件大小 (e.g., 2.3MB)
- ✅ 登记时间
- ✅ 你的名字 (来自系统设置)

---

### 🎬 生视频组 - 图片到视频关联

**目标**: 记录哪个视频是由哪个图片生成的

**操作步骤**:

#### Step 1: 先登记原始图片（同生图组）
1. 素材登记 → 拖入原始图片 → 登记

#### Step 2: 登记生成的视频
1. 素材登记 → 拖入生成的视频 → 登记

#### Step 3: 建立关联
1. 切换到 **"处理关联"** 表签
2. 左侧"引用前序素材": 拖入 **原始图片**
3. 右侧"修改后新素材": 拖入 **生成的视频**
4. 点击 **"建立溯源关联"**

**系统自动记录**:
- ✅ 图片hash → 视频hash 的关联
- ✅ 你是修改操作员
- ✅ 操作时间
- 关系类型自动判断为 `image_to_video`

**示例**:
```
source: IMG_20260310_001.png (hash: a1b2c3d4...)
   ↓ (modified by 王芳)
target: VIDEO_OUT_20260310.mp4 (hash: x9y8z7w6...)
```

---

### 🎨 Canva视频组 - 成品合成

**目标**: 记录Canva视频用了哪些图片/视频，最后生成的成品是什么

**操作步骤**:

#### Step 1: 下载素材后登记
1. 素材登记 → 拖入所有下载的原始素材
   - 比如: `stock_image_1.png`, `stock_video_1.mp4`
2. 登记它们

#### Step 2: 制作Canva视频
（在Canva中正常制作，导出为MP4）

#### Step 3: 制作完成后，在软件中记录

1. 切换到 **"成品封装"** 表签

2. 左侧"组件素材" (多选):
   - 拖入所有使用过的图片/视频
   - 例如: `stock_image_1.png`, `stock_video_1.mp4`

3. 右侧"最终成品" (单选):
   - 选择 Canva导出的 `canva_output_20260313.mp4`

4. 其他字段可选:
   - "手动补链" 可以补充任何备注 (JSON格式最佳)
   - 例如: 
   ```json
   {
     "canva_settings": "background: dark",
     "notes": "使用了字体：微软雅黑"
   }
   ```

5. 点击 **"封装成品"**

**系统自动**:
- ✅ 关联所有组件素材 (图片1, 视频1, ...) → 成品
- ✅ 合并所有元数据
- ✅ 记录你的名字和操作时间
- ✅ 如果成品是PNG，自动写入metadata

**查询结果**:
- 审计人员查询该Canva视频时，会看到完整链路：
  ```
  Canva成品 (hash: final_xxx)
  ├─ 使用第1个素材: stock_image_1.png
  ├─ 使用第2个素材: stock_video_1.mp4
  └─ 创建人: 你的名字 (时间)
  ```

### ✂️ 剪辑视频组 - 成品制作

**目标**: 完全同 Canva 视频组

**操作流程完全相同**:
1. 登记下载的原始素材
2. 在剪辑软件中制作
3. 成品封装 → 记录所有使用的素材
4. 系统自动记录关系

**区别**: 最终成品是 `.mp4` (剪辑版本) 而非 Canva输出

---

### 🔍 审计人员 - 查询溯源

**目标**: 验证一个视频/图片的来源和制作链路

**操作步骤**:

1. 切换到 **"审计看板"** 表签

2. 拖入待审计文件:
   - 可以是单个文件
   - 也可以拖整个文件夹 (会自动扫描)

3. 点击 **"启动比对"**

4. 查看结果表格:
   ```
   状态      | 文件名                | 原作者    | 相似度  | 原始名
   ✅已识别   | video_to_check.mp4    | 王芳      | 95%     | canva_output.mp4
   ```

5. 如果发现问题:
   - 查看"全量库"了解所有登记素材
   - 手动查询数据库获得更详细信息

**审计报告要点**:
- ✅/❓ 图标: 是否在库中找到
- 相似度: 哈希匹配度 (≥95% 认为是同一文件)
- 原始名: 库中记录的原名
- 原作者: 谁上传/创建的

## 🛠️ 系统设置

1. 切换到任何表签，点击 **"⚙️ 系统设置"** 按钮

2. 可以修改:
   - **审计员姓名**: 你的名字 (会显示在所有操作记录中)
   - **MySQL 地址**: 数据库服务器 (默认localhost)
   - **MySQL 密码**: 数据库密码
   - **数据库名**: 库名 (默认 mam_system)

3. 点击 **"Ok"** 保存

---

## 📊 "全量库" 表签

查看系统中所有已登记的素材:

**列**:
- 文件名: 原始文件名
- 作者: 谁登记/创建的
- 时间: 登记/创建时间
- pHash: 素材的哈希值 (前16位)

**用途**:
- 快速查找素材
- 验证素材是否已登记
- 审计检查

## 💾 文件位置

所有记录保存到:
- **配置**: `mam_config.json` (你的用户名)
- **数据库配置**: `mam_db_config.json` (MySQL连接信息)
- **MySQL数据库**: 三个核心表
  - `assets`: 所有素材记录
  - `asset_relations_11`: 一对一关联 (修改链)
  - `asset_relations_nm`: 一对多关联 (成品链)

## ⚠️ 故障排查

### 问题: "❌ 数据库警告: ..."

**原因**: MySQL连接失败

**解决**:
1. 确保 MySQL 服务已启动
2. 打开"⚙️ 系统设置"检查:
   - MySQL 地址是否正确 (如果本机用 localhost)
   - 密码是否正确
   - 数据库名 `mam_system` 是否存在

```sql
-- 如果数据库不存在，在MySQL中执行：
CREATE DATABASE mam_system CHARACTER SET utf8mb4;
```

### 问题: 拖入文件后没有显示预览

**原因**: 文件格式不支持或路径有特殊字符

**解决**:
- 确保是 `.png`, `.jpg`, `.mp4` 等支持的格式
- 文件名避免特殊符号

### 问题: 素材找不到或相似度很低

**原因**: phash计算差异大 (例如压缩等级变化)

**解决**:
- 系统默认相似度阈值是 81% (可找到基本同源素材)
- 如果误差超过此值，需要在DB中手动调整
