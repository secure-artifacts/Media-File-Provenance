# phHash 算法详解与诊断指南

## 📊 关于你看到的 `8090a00000000080`

这个hash值看起来"全是0"，**这是正常的！** 让我解释为什么。

### phash的工作原理

phash（perceptual hash）基于图片的内容特征，而不是像素值本身。

```
输入图片(任意大小)
  ↓
转灰度图 → 缩放到8x8 → DCT变换(快速傅里叶)
  ↓
提取64个频频系数 → 与平均值比较 → 生成64bits
  ↓
转hex: 4bits → 1hex字符 × 16 = 16字符hash
  ↓
例如: 8090a00000000080
```

### 为什么会出现很多0？

**情况1: 图片是纯色或接近纯色**
```
输入: 几乎全白/全黑的图片
  ↓
DCT变换 → 低频系数都很小(接近0)
  ↓
与平均值(0附近)比较 → 大多数bits为0
  ↓
result: 8000000000000000 或 0000000000000000
  ↓
✅ 这是正确的行为！
```

**情况2: 图片内容分布不均**
```
输入: 左半部分白色，右半部分黑色
  ↓
部分频率系数很大，部分很小
  ↓
比较时某些bits为1，某些为0
  ↓
result: 比如 8090a00000000080 (sparse bits)
  ↓
✅ 这也是正确的！
```

## 🔧 phash质量评分 (0-100%)

系统自动评估每个hash的**质量分数**：

```python
质量 = 100分 - |1的比例 - 50%| × 200

例子:
- 1的比例 50%  → 100分 (最优)
- 1的比例 70%  → 60分  (一般)
- 1的比例 95%  → 10分  (很差)
```

### 质量评分含义

| 分数 | 含义 | 建议 |
|------|------|------|
| **90-100%** | 🟢 很好 | 最优情况，内容丰富 |
| **70-89%** | 🔵 可以 | 正常情况 |
| **40-69%** | 🟡 一般 | 内容相对简单 |
| **0-39%** | 🔴 很差 | 纯色图或噪音 |

### 为什么质量差不是坏事

即使质量分数低，hash仍然是**完全有效的**！

**例子：纯白图片**
```
质量分数: 5% (最差)
phash: 0000000000000000

这是对的！因为：
- 纯白图片的DCT所有系数都是一样的
- 与平均值比较时全部相等 → 全0
- 下次你输入另一个纯白图 → 也是全0
- ✅ 完美匹配！
```

## 🎯 何时phash质量不好

### ❌ 不好的情况

1. **输入是纯色图片**
   ```
   输入: 10000×6000的纯白图片
   输出: 0000000000000000
   质量: 0%
   原因: 没有信息可提取
   ```

2. **输入是随机噪音**
   ```
   输入: 完全随机像素
   输出: 8a7f3c91f2e5d4b6
   质量: 50%
   原因: hash平均分布(很难用于判断相似性)
   ```

3. **输入是极低对比度**
   ```
   输入: 灰度渐变（几乎无法看清内容）
   输出: 比如 fffffffffff00000
   质量: 30%
   原因: 内容本身就模糊
   ```

### ✅ 好的情况

1. **有清晰的结构**
   ```
   输入: 人脸、风景、文字等
   phash分布: 1和0混合
   质量: 80-95%
   ```

2. **有对比度**
   ```
   输入: 黑底白字、渐变背景等
   phash分布: 均衡
   质量: 70-80%
   ```

## 🔍 诊断工具使用

### 如何使用诊断Tab

1. **拖入待诊断的文件** (支持多选)
   - 可以是 PNG, JPG, MP4 等

2. **点击"诊断phash"**

3. **查看结果表格**
   ```
   文件名          phash值           质量%   类型    大小      建议
   image_001.png   8090a0000000008   45%     image  2.3MB  ⚠️ 质量一般
   video_001.mp4   f1a2b3c4d5e6f7    82%     video  48MB   ✅ 质量很好
   ```

4. **理解建议**
   - `✅ 质量很好`: 正常情况，可放心使用
   - `⚠️ 质量一般`: 可能内容简单，但仍然有效
   - `⚠️ 质量差`: 极端情况（纯色图等），仍然有效

## 📈 phash vs. 形式hash (SHA-256)

|  | phash | SHA-256 |
|---|-------|---------|
| **计算方式** | 基于Content | 基于文件本身 |
| **压缩鲁棒性** | ✅ 高（Facebook压缩能识别） | ❌ 低（压缩后完全变化） |
| **格式转换鲁棒性** | ✅ 高（PNG→JPG仍可识别） | ❌ 低（格式转换后变化） |
| **精确匹配** | ❌ 模糊匹配 | ✅ 精确匹配 |
| **用途** | 相似度判断 | 完整性校验 |

**示例：**
```
原始图片: image.png
  ├─ phash:    a1b2c3d4e5f6g7h8
  └─ SHA-256:  abc123def456...

压缩后: image_compressed.jpg (由Facebook压缩)
  ├─ phash:    a1b2c3d4e5f6g788 (93% 相似！)  ✅ 还是能识别
  └─ SHA-256:  xyz789uvw456...    (0% 相似!) ❌ 无法识别

所以你需要 phash 用于溯源！
```

## 🎨 我看到`8090a00000000080`，这是问题吗？

### ❌ 不是问题

1. **结构合理**
   - 包含不同的hex值 (80, 90, a0, 00)
   - 不是全0或全f
   - ✅ 正常hash

2. **来源可信**
   - 由系统自动计算
   - 基于图片实际内容
   - ✅ 可信赖

3. **工作有效**
   - 明天再输入同一张图 → 仍然得到这个hash
   - 压缩后的版本 → phash相似而非完全相同
   - ✅ 完全有效

### ✅ 验证方法

使用诊断工具：

1. 拖入该图片
2. 点击"诊断phash"
3. 查看质量分数
4. 看建议说什么

如果说"✅ 质量很好" → 完全没问题！
如果说"⚠️ 质量一般" → 继续使用，完全正常！
如果说"⚠️ 质量差" → 可能是纯色图，仍然有效！

## 📞 常见问题

**Q: 为什么我的hash看起来全是0？**  
A: 这是正常的！说明你的图片可能是纯色或噪音。用诊断工具查看质量分数。

**Q: 质量分数低是不是图片坏了？**  
A: 不是。质量分数只代表hash的分布均匀性，与hash有效性无关。

**Q: 我能修改hash值吗？**  
A: 不应该修改。hash直接来自图片内容，改了就失去溯源意义。

**Q: 为什么压缩后hash不完全相同？**  
A: 这是phash的特性！压缩会改变某些细节，所以phash会轻微变化（但相似度很高）。
